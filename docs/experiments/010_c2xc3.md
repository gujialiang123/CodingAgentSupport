# Experiment 010 — C2 × C3 (helper tests × gates) 2×2

**Plan priority 4.** Isolate the individual and joint effects of C2 (helper
reproduction tests) and C3 (validation gates), now that P1 (container execution)
and P2 (container-validated, frozen T3/T4 helpers) make both supports real.

## 1. Design

2×2 factorial, one factor per axis:

|            | no Gates | Gates |
|------------|----------|-------|
| no Helper  | C0_minimal | C3_gates |
| Helper     | C2_tests   | C2_C3 |

- **Cohort:** `data/tasks/ablation_t34.jsonl` — the **7/12 tasks** whose C2 helper
  is class **T3/T4** (container-validated fail-before + pass-after-gold), frozen
  in `data/helpers/`. Using only T3/T4 tasks removes the confound of invalid
  helpers so C2's effect is measured where the helper actually reproduces the bug.
- **Seeds:** 0,1,2 (3 seeds).
- **Runs:** 7 tasks × 4 conditions × 3 seeds = **84 runs**.
- **Budget:** 25 turns, in-container (`--in-container`), qwen3.7-plus, max_tokens 4096.
- **Helpers:** loaded read-only from the frozen cache (`--helper-cache-dir
  data/helpers`); no per-run regeneration, so the exact validated T3/T4 helper is
  what the agent sees, and C2/C2_C3 are deterministic w.r.t. the helper.

The 7 T3/T4 tasks: psf/requests-1142 (T4), psf/requests-2931 (T3),
pytest-7571 (T4), pytest-10051 (T4), pylint-4604 (T3), flask-5014 (T4),
pydata/xarray-3677 (T4).

## 2. Questions this cleanly answers

1. Do **helper tests** (C2) raise resolution / F2P over C0?
2. Do **gates** (C3) raise resolution or mainly cut P2P regressions over C0?
3. Are helper and gates **complementary** (C2_C3 > max(C2,C3)) or redundant?
4. Does C2 raise F2P but induce **test overfitting** (helper passes, official fails)?

## 3. Test-generation feasibility (P2 result, reported, not dropped)

From freezing helpers on the full 12-task cohort (qwen3.7-plus, k=3, in-container):

| Class | n | meaning |
|-------|---|---------|
| T4_decoupled_valid | 5 | T3 + no leakage/overfit audit flag |
| T3_valid_reproduction | 2 | fails on base, passes on gold |
| T2_incompatible_oracle | 2 | fails on base AND on gold |
| T1_non_reproducing | 1 | passes on base (no repro) |
| T0_invalid | 2 | cannot collect/execute |

**7/12 (58%) usable (T3/T4).** This is itself a headline feasibility number:
a blind, gold-free generator produces a container-valid reproduction test for the
majority of issues, but ~42% are unusable — so C2 must be reported conditional on
helper validity, which is exactly what this cohort does.

## 4. Results

> **⚠️ ERRATUM (protocol 0.3.0):** the raw numbers below were contaminated by
> helper-in-patch and base-image `build/lib` leakage. See
> [`010_c2xc3_erratum.md`](010_c2xc3_erratum.md) for the **corrected** values
> (C0 0.52, C2/C3/C2_C3 0.67; files_touched ~1.0–1.4; over-editing claim
> withdrawn). The qualitative conclusions below still hold; the absolute
> resolution and locality numbers are superseded by the erratum.

84 runs complete (`exp010_c2xc3`, 7 T3/T4 tasks × 4 conditions × 3 seeds,
in-container, 25 turns, frozen helpers). All cells ran cleanly (0 infra errors).

| Condition | n | Resolved | Q2 | P2P regression* | mean files touched |
|-----------|---|----------|----|-----------------|--------------------|
| C0_minimal (no helper, no gates) | 21 | **0.38** | 8 | 4/14 (29%) | 1.1 |
| C2_tests (helper) | 21 | **0.52** | 11 | 3/15 (20%) | 2.0 |
| C3_gates (gates) | 21 | **0.52** | 11 | 4/15 (27%) | 1.3 |
| C2_C3 (helper+gates) | 21 | **0.52** | 11 | **2/15 (13%)** | 1.9 |

*P2P regression = share of applying patches that break ≥1 previously-passing
test. Per-seed resolution is stable (C0 3/3/2, C2 3/4/4, C3 4/3/4, C2_C3 4/4/3).
Gates fired in both gated conditions (7 blocking failures → 7 revisions each).

### Findings

1. **Both C2 and C3 individually lift resolution 0.38 → 0.52 (+0.14)** and the
   effect is stable across 3 seeds — helper tests *and* gates each help the agent
   land a correct patch on this cohort/model.
2. **They do not stack on resolution:** C2_C3 is also 0.52, not higher than either
   alone. On these tasks helper and gates are largely *redundant for correctness*
   — each supplies enough signal to fix the reachable bugs; combining them adds no
   new correct solves.
3. **The joint benefit shows up in patch *safety*, not resolution.** C2_C3 has the
   lowest P2P regression rate (13%) vs C0 (29%), C2 (20%), C3 (27%). Gates catch
   regressions the helper doesn't, so helper+gates yields the safest patches even
   when it doesn't yield more of them.
4. **C2 broadens edits** (2.0 files touched vs C0 1.1): a runnable reproduction
   test pushes the agent to change more code — a mild over-editing tendency to
   watch for test-overfitting on larger cohorts.
5. **Interpretation for the study:** report C2 and C3 as *both individually
   effective but overlapping*; the value of stacking them is a **quality/safety**
   effect (fewer P2P regressions), which is exactly the kind of claim the 2×2
   cleanly supports. Resolution-only reporting would have missed this.

### Caveats
- n=7 tasks × 3 seeds; directional. The 7 tasks are exactly those with a
  container-valid (T3/T4) helper, so C2's effect is measured where the helper
  actually reproduces the bug (by construction, not cherry-picking).
- Single model (qwen3.7-plus). Stronger models may show different C2/C3 overlap.
